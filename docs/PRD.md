# 产品需求文档

## 产品名称

- 中文：`灵演`
- 英文：`WonderShow`

## 产品定位

灵演是一款面向演讲、课程录制、直播和排练场景的 macOS 演示导演台。它负责把摄像头预览、屏幕/窗口采集、演示控制、录制流程、项目保存、后期合成和手势输入整合为一套低学习成本的工作流。

## 核心用户

- 需要边讲边翻页的讲师
- 需要录制课程或演示视频的内容创作者
- 需要连接外接相机和大屏的直播/活动演讲者

## 当前问题

- 录制工作室主链路已经可用，但底部时间轴仍是状态展示，不是真正可编辑的时间轴。
- 活动窗口进入监视器后存在抖动问题，需要稳定窗口捕获和预览布局。
- 讲者画面还缺少美颜、调亮、镜像翻转等质量增强能力。
- App 还不是 macOS 菜单栏常驻工具，也没有可在桌面自由拖拽的录制 mini toolbar。
- 手势识别仍需从静态/近距离识别升级到远距离动态动作识别。
- 商业化分发和授权验证机制尚未设计，未来上架 App Store 或直营网售前需要补齐。
- 当前仍是 macOS 单端产品，多端点支持、移动设备协同和跨平台项目格式还未设计。
- 当前只有暖黑金暗色视觉，未来需要多皮肤和 light/dark 主题能力。

## 本阶段目标

- 固化 `v0.7.20260618` 录制工作室阶段基线。
- 保持已通过复测的录制、选源、画中画、预览、导出、收音能力稳定。
- 下一轮优先修复监视器活动窗口抖动，并增强讲者画质、时间轴编辑和录制时的轻量控制入口。
- 中期补齐授权验证/付费激活、多端点支持和多主题系统设计。

## 非目标

- 当前阶段不引入云端服务。
- 当前阶段不引入专用硬件作为手势识别前提。
- 当前阶段不把底部时间轴一次性做成完整 NLE，只按可验证步骤推进。
- 当前阶段不重写已经测试通过的录制主链路。

## 成功标准

- 用户可以选择屏幕/窗口、摄像头和麦克风，完成一段录制并保存为 `.wondershow` 项目。
- 用户可以在监视器中拖拽/缩放/改变画中画形状，并在预览与导出中得到一致结果。
- 用户可以预览合成视频并导出真实可播放的 mp4 文件。
- 用户可以在录制中暂停、继续、终止，并在终止后选择保存或放弃。
- 后续可以在不破坏现有录制能力的前提下继续扩展时间轴、美颜、菜单栏/mini toolbar、授权商业化、多端点、多主题和手势模型。
