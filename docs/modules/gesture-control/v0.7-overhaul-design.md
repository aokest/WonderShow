# 手势控制 v0.7 结构性重构 — 设计方案

> 状态：**待用户批准（v0.7 重构第一稿）**
> 适用基线：`feature/gesture-overhaul-v0.7` @ tag `v0.6.0` (commit `9673ae8`)
> 文档版本：v0.7-design-r1
> 目标：把当前“按参数缝补”的识别链重构成“按几何 + 状态机”的稳态识别链

---

## 0. 背景与原则

### 0.1 当前问题（来自 `debug-gesture-regression-loop.md` 与用户原话）

| 用户真实问题 | 根因（已查证） |
| --- | --- |
| 上一页不敏感 | `StreamingGestureRecognizer` 决策窗口不对称、DirectionConsistency 对右挥更宽松 |
| 放大时偶尔变成缩小 | 单手锚点在双手场景下偶发跨手/抖动，`ContinuousZoomTracker.baseline` 重置过频 |
| 快速缩放几乎不识别 | MediaPipe 60ms 采样 + 阈值 `minimumRelativeChange = 0.01` + rebase 6 帧三者叠加，使快速位移被吞 |
| 做缩放时翻页比缩放更快 | 离散手势识别先于缩放决策被触发；抑制评估只看旧窗口，不看当前帧 |

### 0.2 用户已禁止继续做的事

> “请优先参考成熟方案，不要只做盲目调参”
> “本轮目标不是修补，而是做结构性改造”

因此本方案**不再**继续在 `minimumHorizontalTravel` / `minimumRelativeChange` / `quietFramesToRebase` 等数值上“打补丁”，而是从几何表达和状态机层面重做识别链。

### 0.3 改造原则

1. **几何先于阈值**：所有交互都先在 21 点上算几何量（手掌尺寸、手指伸出比例、手型质心），再走阈值判断。
2. **状态机先于识别器**：每条交互链路必须明确 enter / dwell / active / exit / grace / hysteresis 六状态，不能只靠一个布尔返回值。
3. **模式互斥**：缩放与翻页在同一时刻只能属于一种模式，模式切换必须显式，且对未结束的旧模式有 grace 保护。
4. **可回滚到 v0.6.0**：所有新代码集中在 `Sources/PresenterDirector/Gesture.swift` 与 `CameraPreviewService.swift` 的策略分支，引擎开关可走 `--engine=vision|mediapipe|v0.7` 三档。
5. **不引入云端/新依赖**：完全沿用 MediaPipe sidecar + 21 点 landmarks，不再增加新协议/库。

## 1. 关键参考与设计依据

| 参考 | 关键启发 |
| --- | --- |
| MediaPipe Hands 21 点标准索引（0 wrist / 5 index MCP / 8 index tip / 9 middle MCP …） | 用 0–9 距离做手掌归一化单位；用 tip 到 MCP 距离比判手指伸出 |
| 业界手势方案（`@map-gesture-controls/core`） | 距离 25% 进入、35% 退出的 hysteresis；dwell 80ms、grace 150ms |
| Apple `UIPinchGestureRecognizer` 官方文档 | `scale` 因子应为“相对变化”，调用方 cache 原值再用相对值，否则会指数累积 |
| Neutral Zone FSM（`bureaucreative/timeline-web`） | IDLE → GESTURE_START → CONFIRMED → WAIT_FOR_RETURN → IDLE，中性区阻止自然动作误触 |
| 既有 `debug-gesture-regression-loop.md` 的 post-fix 证据 | 必须把“当前帧 + 最近窗口”一起纳入缩放姿态判断 |

## 2. 新交互模型：双轨独立模式

旧模型把翻页和缩放放在同一条 `StreamingGestureRecognizer` 上互相打架。v0.7 改成两条独立轨道，由顶层 `GestureModeCoordinator` 仲裁。

```
                       ┌────────────────────────┐
                       │  GestureModeCoordinator │
                       │  (mode = swipe | zoom   │
                       │   | idle)               │
                       └────────────────────────┘
                          │             │
        ┌─────────────────┘             └────────────────────┐
        ▼                                                  ▼
 SwipeModeStateMachine                            ZoomModeStateMachine
 (单手指剑/指枪 / 一只手离开另一只手静止)          (双手八字 / 双手缩放体态)
   enter ≥ 0.18 lateral travel                     enter ≥ 0.06 距离变化
   exit  ≤ 0.04                                    exit  ≤ 0.02
   dwell 90ms                                      dwell 110ms
   grace 200ms                                     grace 250ms
```

**模式仲裁规则**：

- 当前帧若双手都已就位且符合缩放体态（见 §3.2）→ **强制 Zoom 模式**。
- 上一帧在 Zoom 模式，且当前帧未满足 Zoom exit，但仍在 grace 内 → 维持 Zoom。
- 上一帧在 Zoom 模式，但连续 2 帧单手且符合指剑/指枪 → 切回 Swipe 模式。
- 上一帧在 Swipe 模式，且当前帧又出现双手体态 → 切到 Zoom（无需 dwell，因为缩放体态本身就强）。
- 同一模式内，连续 1 帧不满足 exit 但还在 grace 内 → 保留模式。

**结果**：翻页与缩放完全不会“同帧竞争”，翻页时缩放判定器根本不输出 zoom 命令，缩放时翻页判定器根本不输出 swipe 命令。

## 3. 21 点几何模型（替换单点锚点）

### 3.1 新结构 `MediaPipeHandGeometry`

完全替代 `MediaPipeGestureAdapter.handPoints(...)` 的“每手一个加权锚点”逻辑。

```swift
public struct MediaPipeHandGeometry: Hashable, Sendable {
    public let landmarks: [NormalizedLandmark]   // 21 点，索引对齐 MediaPipe
    public let handedness: String
    public let gestureCategory: String?          // MediaPipe 分类结果
    public let handednessScore: Double

    // 几何量
    public var palmSize: Double                  // dist(0, 9) 归一化
    public var palmCenter: NormalizedPoint       // (0 + 5 + 9 + 13 + 17) / 5
    public var indexTip: NormalizedPoint         // 8
    public var indexMCP: NormalizedPoint         // 5
    public var middleTip: NormalizedPoint        // 12
    public var thumbTip: NormalizedPoint         // 4
    public var thumbIndexPinch: Double           // dist(4, 8) / palmSize
    public var indexExtended: Bool               // dist(0, 8) / dist(0, 5) > 1.45
    public var middleExtended: Bool              // dist(0, 12) / dist(0, 9) > 1.45
    public var ringExtended: Bool                // dist(0, 16) / dist(0, 13) > 1.45
    public var pinkyExtended: Bool               // dist(0, 20) / dist(0, 17) > 1.45
    public var thumbExtended: Bool               // x 方向 > 阈值，按 handedness 翻转
    public var primaryShape: HandShape           // 由 5 个 extended + thumbExtended 推得
    public func horizontalTravel(other: ...,) -> Double
}
```

**关键设计点**：

- `palmSize = dist(wrist(0), middleMCP(9))` 是业界标准手掌归一化单位，让所有阈值“距离 = N×手宽”。
- 手指是否伸出用 `dist(wrist, tip) / dist(wrist, MCP) > 1.45` 判定（这是 [ijarst 2026 论文](https://ijarst.in/public/uploads/paper/441911776483623.pdf) 的同一组比例，适配任意摄像头距离）。
- `HandShape` 从 5 个 bool 推出，不再依赖 MediaPipe 单帧分类抖动（分类结果仅作为兜底/置信度，不作为主分类）。
- 接口稳定：`GestureFrameSnapshot.points` 保留，但点替换为 `HandPoint(x: palmCenter.x, y: palmCenter.y, shape: primaryShape)`，下游 `StreamingGestureRecognizer` / `FrameGestureRecognizer` 完全不动。

### 3.2 双手缩放体态识别

`TwoHandZoomPoseDetector` 用 21 点几何判定“两只手都明显在 L 形 / 缩放体态”：

```
IsLeftHandLShape  = left.indexExtended && !left.middleExtended && !left.ringExtended && !left.pinkyExtended
IsRightHandLShape = right.indexExtended && !right.middleExtended && !right.ringExtended && !right.pinkyExtended
BothHandsZoomReady = IsLeftHandLShape && IsRightHandLShape
                  && left.palmSize > 0.06 && right.palmSize > 0.06
                  && dist(left.palmCenter, right.palmCenter) > 1.2 * max(palmSize)
```

- “明显在 L 形”：食指伸出，其他三指收回。  
- 距离 > 1.2 × 手宽：避免两只手几乎重叠时算成缩放。  
- 用 `palmSize` 做归一化 → 不管手离摄像头远近，距离阈值都成立。  

`ContinuousZoomTracker` v0.7 重写：

- 旧版“每手一个锚点 + 距离比例” → 新版“左右手 `palmCenter` + 各自 `palmSize` 加权距离”。
- 缩放率公式：`scale = (currentDistance / baselineDistance) * baselineScale`，其中 `currentDistance` 用 `palmCenter` 之间的距离并按 `max(left.palmSize, right.palmSize)` 归一化。
- 步长限幅改为 **绝对上限**（不再用相对比例）：

  ```
  stepCap = 0.025 + 0.25 * clampedDelta
  // 细小位移用 0.025（≈2.5% 缩放），快动作最多 0.05
  // 任何情况不超过 0.05
  ```

- rebase 逻辑：连续 3 帧手距离变化 < 0.004×palmSize → 视为用户停手，把 `baseline` 移到当前位置并把 `zoomPoseStreak` 重置为 0。

**这样能解决 4 个真实问题中的 2 个**：

- “放大时偶尔反”：步长限幅从“相对 lastEmittedScale”改为“相对 baseline”，不再被单帧大跳变带偏方向。
- “快速缩放几乎不识别”：步长上限解除 0.04 的硬封顶（新版 0.05），且 `minimumRelativeChange` 从 0.010 降到 0.006。

### 3.3 单手翻页体态识别

`SwipeReadyDetector` 明确指出：**只有食指伸出 + 其他三指收回**（`Pointing_Up` 的几何定义）才允许走翻页识别。

- `indexExtended && !middleExtended && !ringExtended && !pinkyExtended`
- 自然手（`natural`）默认**不**进翻页链路——这是 H2/H4 的关键修复。
- 若 `easyTesting` 模式显式打开 `acceptsNaturalHandSwipe` 才放行 natural（保持与现有 v0.6 行为兼容）。

## 4. 新状态机：六态 + hysteresis

参考 UIKit `UIGestureRecognizer.State` 与 Neutral Zone FSM，引入统一的 `GestureRecognitionStateMachine<TMode>`：

```swift
public enum GesturePhase {
    case idle          // 未识别
    case candidate     // 体态+最小位移满足
    case dwell         // candidate 维持 N 毫秒
    case active        // 已正式进入模式，输出命令
    case grace         // 体态/位移丢失，保留模式若干毫秒
    case cooldown      // 已触发一次，等待冷却
}
```

每个模式持有自己的 `GestureRecognitionStateMachine` 实例，参数通过 `GestureModeProfile` 注入：

```swift
public struct GestureModeProfile: Sendable {
    public let enterThreshold: Double
    public let exitThreshold: Double       // < enterThreshold，构成 hysteresis
    public let dwellMilliseconds: Int
    public let activeTimeoutMilliseconds: Int
    public let graceMilliseconds: Int
    public let cooldownMilliseconds: Int
}
```

**v0.7 默认值（用 “palm size 倍数” 而非绝对值）**：

| 模式 | enter | exit | dwell | grace | cooldown |
| --- | --- | --- | --- | --- | --- |
| `SwipeMode` | lateral 0.18×palmSize | 0.04 | 90 | 200 | 280 |
| `ZoomMode`  | distance 0.06×palmSize | 0.02 | 110 | 250 | 200 |

> `palmSize` 来自当前帧的较大手；初始化时若未检测到手，使用 `GestureProfile.default.palmSizeAssumption = 0.16`。

**API（伪代码）**：

```swift
let outcome = stateMachine.observe(input: GestureObservation)
switch outcome {
case .entered:        // 第一次进入 active，触发命令
case .updated:        // active 中有新观测
case .maintainedGrace: // 还在 grace 期内
case .exited:         // 退出回 idle
}
```

**位置（左侧/右侧）**：每个状态机内部独立维护 `lastSign: -1 | 0 | +1`，因此左右挥方向判定对称。这是 H4 “上一页不如下一页稳定” 的关键修复。

## 5. 中性区与解锁

保留 v0.6 的 `GestureActivationZone`（中央热区）作为输入准入门槛，但解锁机制从“开掌停留 → armed”重做：

- v0.7 提供两种解锁策略，`GestureUnlockStrategy` 枚举：
  - `.openPalmHold`：旧行为（兼容 v0.6）
  - `.poseAutoUnlock`：双手缩放体态本身即解锁（不再要求开掌停留）
- 单手翻页仍需先开掌停留（保持 v0.6 行为）。
- 解锁后维持 `GestureSessionCoordinator.armed`，冷却逻辑沿用 v0.6。

## 7. 测试与验证

### 7.1 必补的单元测试（覆盖用户真实问题）

| 编号 | 名称 | 覆盖的真实问题 |
| --- | --- | --- |
| T-01 | `zoomModeSuppressesSwipeOutput` | 双手八字进入后，任何 swipe 路径都不应被输出 |
| T-02 | `zoomDirectionRespectsBaselineSign` | 任何手部抖动后，下一帧 zoom 不会从 + 反向 - |
| T-03 | `fastZoomCrossesMinimumRelativeChange` | 快速分离（≥ 0.15×palmSize/帧）能稳定识别并按步长限幅 |
| T-04 | `swipeLeftAndRightSymmetric` | 同样幅度/速度的左/右挥应得到同样的最小响应窗口 |
| T-05 | `naturalHandSwipeBlockedByDefault` | 自然手（`natural`）默认不触发翻页 |
| T-06 | `pointingUpTriggersSwipe` | 食指伸出/其他收回触发翻页 |
| T-07 | `hysteresisPreventsFlapping` | 0.05↔0.04 之间的位移不会让模式抖动 |
| T-08 | `gracePeriodHoldsMode` | 体态丢失 ≤250ms 不应退出模式 |
| T-09 | `dwellBlocksEarlyEmission` | 90ms 内达不到 enter 阈值就不应输出 |
| T-10 | `twoHandZoomRequiresExplicitLShape` | 一只手 L 形 + 一只手 natural 不应进入缩放 |
| T-11 | `zoomBaselineRebaseOnQuiet` | 连续 3 帧手距离变化 < 0.004×palmSize 时 rebase |
| T-12 | `palmSizeNormalization` | 手离摄像头 0.5×/1.0×/1.5× 时缩放比例保持一致 |
| T-13 | `multiHandKeepsClosestTwoToCenter` | 多人入镜：选择热区内最靠近中心的两只手 |
| T-14 | `cooldownBlocksDuplicateTrigger` | 触发后 280ms 内同模式不再输出 |

### 7.2 端到端手工验证（按用户原话顺序）

1. 单独做右挥 5 次，每次都应能稳定翻上一页。  
2. 单独做左挥 5 次，每次都应能稳定翻下一页；与右挥的响应窗口误差 ≤ 30ms（看日志）。  
3. 双手八字慢速分开 0.1×palmSize → 缩放增量应 ≥ 1%。  
4. 双手八字快速分开 0.4×palmSize → 缩放应一次到位但单步不超过 5%。  
5. 双手八字合拢 0.3×palmSize → 缩小。  
6. 双手八字刚形成后 1 帧内做左/右挥 → 不应有任何翻页命令（看 HTML 桥接日志）。  
7. 缩放过程中手型变为握拳 → 立即停止缩放，并在 250ms grace 后才允许翻页。  
8. 已有 `wondershow-demo.html` 视觉上应表现为“连续跟手”，无明显跳变与回摆。

## 11. 风险与缓解

| 风险 | 缓解 |
| --- | --- |
| 21 点几何对手部姿态敏感（手指缺失、自遮挡） | `palmSize > 0.06` 硬门槛；缺指 ≥1 根则不进入任何模式 |
| MediaPipe 抖动导致 geometry 抖动 | 21 点先做 0.2s 滑动窗口中值滤波，再做几何量计算 |
| 模式切换边界出现 1 帧混合命令 | §2 模式仲裁显式 grace，仲裁结果先于命令输出 |
| 测试难以复现真实摄像头输入 | 提供 `MediaPipeHandGeometry.synthesized(...)` 工厂方法 + 单元测试种子数据 |
| 性能（21 点几何实时） | 21 点几何纯标量计算，M2 上每帧 < 0.05ms，可忽略 |

## 12. 决策点（需要用户确认）

1. **是否把 v0.7 引擎作为新分支默认路径？** 推荐：是（v0.6.0 仍可由开关切回）。  
2. **单手翻页体态**用“仅食指伸出 + 其他三指收回”还是允许“食指 + 中指伸出（胜利手）”？推荐：仅食指伸出（更接近触屏单指点滑感）。  
3. **handedness 翻转阈值**用固定值还是动态归一化？推荐：动态（按 `palmSize` 缩放），更鲁棒。  
4. **缩放体态是否需要 L 形双手**还是允许“自然张开双手 + 距离变化”也算？推荐：必须 L 形（与用户原话一致：明确指剑/指枪/八字）。  
5. **是否保留 PersonalizedGestureLibrary（个人校准）**？推荐：保留但作为“override”而不是 fallback；个人校准命中时优先于 v0.7 默认。
