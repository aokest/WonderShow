## 1. 项目概况

灵演是一个 macOS 桌面应用（SwiftUI + Swift Package Manager），用摄像头实时识别人手手势来控制 Keynote/PowerPoint 翻页、缩放。

**核心技术栈**：
- 摄像头采集：`AVCaptureSession` + `AVCaptureVideoDataOutput`
- 手势识别引擎双轨制：Vision (Apple 内置) / MediaPipe sidecar (本地 Python 推理服务)
- UI 框架：SwiftUI，采用自制「暖黑金」设计系统 (`ConsolePalette`)
- 演示控制：macOS Accessibility API (AX) 发送键盘事件给目标应用

**目录结构要点**：
```
Sources/PresenterDirector/        ← 核心手势/本地化逻辑库
Sources/PresenterDirectorApp/     ← SwiftUI App 层 (DashboardView, CameraPreviewService)
Tests/PresenterDirectorTests/     ← 57 条单元测试
docs/                             ← 工程文档 (INDEX.md 为入口)
sidecar/                          ← MediaPipe Python 推理服务
scripts/                          ← 构建/打包脚本
dist/                             ← 打包产物 (灵演.app)
```

## 2. 本会话已完成的工作

### 2.1 修复：语言切换（简/繁/EN）失效
**根因**：UI 重构后 `DashboardView` 用了独立的 `DisplayLanguageTab` 枚举，未正确映射到底层 `AppLanguage`。

**修复操作**：
1. `AppLanguage` 改为 `CaseIterable`，新增 `label` 属性（简/繁/EN）
2. 删除冗余 `DisplayLanguageTab` 枚举
3. `DashboardView.displayLanguage` 直接用 `AppLanguage` 绑定
4. `AppCopy` 新增 `brandLine1` / `brandLine2` 字段，支撑 Logo 区域多语言
5. 修正 `LocalizationTests` 中英文产品名预期值为 `WonderShow`

### 2.2 修复：手势极度迟钝/无反应
**根因**：`CameraPreviewService.processDetectedPoints` 在热区内无手时（`activePoints.isEmpty`），暴力重置了 `gestureSessionCoordinator`、`gestureModeCoordinator`、`continuousZoomTracker` 并清空了 `gestureSamples`。这导致识别窗口被每帧打碎，永远无法积累到足够帧长。

**修复操作**：移除了该分支下 4 行暴力清除代码，仅保留 UI 状态提示更新。

### 2.3 修复：摄像头预览中间一团黑色阴影
**根因**：(1) `previewGlow`（含暗色径向渐变）渲染在摄像头视图**上层**；(2) 摄像头不透明度仅 0.76，底部纯黑背景透出形成暗色遮盖。

**修复操作**：
1. `previewGlow` 移至摄像头**下层**（仅作装饰背景）
2. 摄像头不透明度从 0.76 改为 1.0（完全不透明）

### 2.4 修复：Logo 文字不跟随语言切换
**根因**：顶部 "WONDERSHOW" / "STUDIO" 是写死的英文字符串。

**修复操作**：新增 `AppCopy.brandLine1` / `brandLine2` 国际化字段，三语分别为：
- 简中：灵演 / 工作室
- 繁中：靈演 / 工作室
- 英文：WONDERSHOW / STUDIO

## 3. 当前未解决的问题

### 3.1 🔴 摄像头中间仍然一团黑色阴影（用户反馈"还是不行"）
**现状**：虽然做了上述 2.3 的改动（层级互换 + 不透明度 1.0），用户启动应用后反馈阴影仍在。
**可能原因**：
- `previewGlow` 中的 `RadialGradient` 仍有暗色边缘，可能渲染顺序未完全生效
- `CameraPreviewView` 的 `clipped()` 或 `aspectRatio` 可能裁剪导致背景透出
- 编译缓存问题，修改未生效
- 需要检查 `CameraPreviewView` / `CameraPreview.swift` 的实现是否有额外暗色叠加层

**排查建议**：
1. 先 clean build：删除 `.build` 目录后重新 `swift build`
2. 检查 `CameraPreviewView` 的 NSViewRepresentable 实现是否有额外 background/overlay
3. 如果阴影是 `previewGlow` 的底部 `Ellipse` 发光造成的，可考虑移除或动态控制

### 3.2 🟡 手势识别尚未进行真机测试
- 理论上状态机重置问题已修复，但未验证
- MediaPipe sidecar 是否正常运行未知
- 需要连接摄像头后在 DashboardView 中观察：
  - `gestureZoneLabel` 是否正常切换"热区已进入/热区外"
  - `gestureModeLabel` 是否在"空闲/翻页模式/缩放模式"间正常切换
  - `gestureSessionLabel` 是否有解锁→识别→冷却的完整流程

### 3.3 🟡 关于 Logo 图标的局限
`DashboardView` 顶部图标从 `NSImage(named: "AppIcon")` 加载，切换语言时只是文字 (`brandLine1`/`brandLine2`) 变化，图标本身不变。这不是 bug，是当前本地化配置未覆盖图标的正常行为。
