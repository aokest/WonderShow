# 灵演 1.0.0 (20260620181913) 版本说明

## 当前定位

`1.0.0` 是核心录制主链路稳定基线。当前优先保护：

- 活动窗口直接采集
- 录制中切换录制源
- 监视器完整窗口显示
- 预览合成与导出主链路
- 高清导出

这些能力已经写入 `docs/CORE_RECORDING_CONTRACT.md`，后续功能不得破坏。

## 本次收口

本版本曾尝试加入讲者高级美颜、背景虚化/替换、Emoji 替脸等实验功能，但真实复测表现不达标：

- 录制时无法可靠实时预览最终效果
- 导出后背景虚化与替换不稳定
- Emoji 替脸无法稳定跟随人脸
- 高级美颜开启后体感接近无效，且早期实时预览路径曾引发严重内存风险

因此当前已将这些实验功能通过 `PresenterExperimentalEffectsGate` 统一禁用：

- 美颜入口置灰
- 背景虚化/背景分割入口置灰
- Emoji 替脸入口置灰
- 即使旧项目或旧 UI 状态残留开关，导出参数也会被清零

亮度、对比度、镜像等基础稳定讲者调整仍保留。

## 当前困境

这组三类效果不能继续用“看起来有按钮”的方式交付。真正可用前，至少需要解决：

1. 录制中监视器必须实时展示与导出一致的结果。
2. 人像分割必须贴合人体轮廓，不能用大圆形或粗糙估算 mask。
3. Emoji 替脸必须跟随人脸位置、尺度和开关状态。
4. 美颜必须有可感知但自然的效果，并且不能产生方块脸、脖子块、过曝或巨大内存占用。
5. 每一项效果都需要真实样例视频和自动化回归测试后才能重新开放。
