Design a macOS-native "Director Console" UI for WonderShow (灵演) — a presentation + recording tool. Single page, dark theme, 1440×900 desktop canvas.

Product positioning: a calm, pro-grade control room for a single speaker who gives presentations, records courses, or runs livestreams. The user must see camera preview, current state, and the next 3 actions in 5 seconds. NOT a marketing site, NOT a flashy AI demo.

=== VISUAL DIRECTION ===
Style: "Broadcast control room × editorial minimalism". Think OBS / DaVinci Resolve / Logic Pro's quieter sibling. Calm, technical, slightly warm. NOT cyberpunk, NOT glassmorphism, NOT neon, NOT gradient mesh, NOT purple-blue AI glow.

Background: #0B0D10 (base) → #14171C (raised surface) → #1B1F26 (overlay). Subtle 1px hairline dividers in #232730 at 60% opacity. Avoid pure #000.

Text: primary #E8E9ED, secondary #9CA0A8, tertiary #5E636C. NEVER pure white. Body 13-14px / line-height 1.5. Generous letter-spacing on caps labels (+4% tracking).

Accent palette (used sparingly, only for state):
- Live/Connected: #3FB6BD (muted teal, broadcast-console feel) — NOT cyan, NOT neon
- Recording: #D8463E (warm red, classic REC dot) — only on the record button + record status
- Warning: #D4A14A (amber, low saturation) — only for permission/connectivity issues
- Success: #5BA873 (muted sage) — only for "calibration complete" style
- Hand-point overlay: #E8E9ED at 70% opacity, 2px stroke, 5px diameter

Typography: Inter for UI (weights 400/500/600/700), JetBrains Mono for timecode / device ID / status values (weight 500). Pairing rule: tabular numerals everywhere, uppercase + tracking for labels > 10 chars, sentence case for everything else.

Corner radius: 12px (cards), 8px (inputs/buttons), 6px (chips). Never below 6.
Shadows: 0 1px 0 rgba(255,255,255,0.04) inner highlight + 0 8px 24px rgba(0,0,0,0.4) for floating elements. NO glow, NO colored shadow.
Borders: 1px solid #232730 always. NO double borders, NO thick dividers.
Motion: 150ms ease-out for hovers, 240ms cubic-bezier(0.2, 0.8, 0.2, 1) for panels. No bouncy springs, no parallax.

=== LAYOUT — single screen, 3 horizontal bands ===

[ Band 1 — Top bar · 56px tall · full width ]
LEFT: 28×28 app mark (rounded square, monogram "灵" in Inter 600) + product name "WonderShow" 15px/600 + a 1px-wide vertical divider + subtle env tag "STUDIO" in JetBrains Mono 11px caps tracking +0.08em, color #9CA0A8.
CENTER: 4 status pills in a single row, equal width, 8px gap. Each pill: 14px icon (Lucide stroke 1.75) + label (11px caps #9CA0A8) + value (13px/500 #E8E9ED). Pills: Camera / Gesture / Target App / REC. REC pill has a 6px pulsing red dot when active. Other pills show a 6px hollow ring when idle, filled teal dot when active.
RIGHT: 2 primary buttons, 32px tall, 8px gap:
  - "Rehearse" — secondary style, #1B1F26 bg, 1px #2D323B border, text #E8E9ED
  - "REC" — primary style, #D8463E bg, white text 13px/600, with a 6px white inner dot. Becomes "STOP" with rounded-square icon when active.

[ Band 2 — Main workspace · fills remaining height · 24px outer padding · 16px gap ]
LEFT COLUMN (60% width, 16:9 preview as anchor):
  - Preview frame: 16:9, 12px radius, 1px #232730 border, black inner bg. Top-left overlay: tiny live indicator (6px teal dot + "LIVE" in 10px caps + tracking 0.1em) at 12px from edge. Bottom-left overlay: 3 stacked chips with translucent #000 60% bg + 8px radius, each chip is 12px icon + 12px text. Chips: gesture zone state, last action ("→ Next slide · 0.3s ago"), detected hand shape.
  - Hand landmark overlay: 21 dots, 5px diameter, #E8E9ED at 70% with 1.5px white stroke. 1px dashed gesture zone rectangle (8 6 dash pattern) at 30% opacity teal when active, gray when idle.
  - Below preview (24px gap): a single horizontal "Director summary" strip — 4 inline cells separated by 1px vertical dividers. Each cell: 11px caps label + 14px/500 value. Cells: Recording mode (Camera only / Camera + Screen) / Layout (PiP · bottom right) / Output (2 tracks) / Elapsed time (00:00:00, JetBrains Mono). NO big tiles, NO big icons here — just typographic facts in one line.

RIGHT COLUMN (40% width, 16px gap, single vertical stack of 4 cards, equal height):
  Card 1 — "Quick start": 4 detail lines (label · value) showing rehearsal state, recording state, active device, current hand shape. Bottom: 2 secondary buttons in a row, "Refresh devices" and "Test slide (3s)".
  Card 2 — "Presentation": 3 form rows, each row = 11px caps label on top + control below. Rows: Target app (segmented chips: PowerPoint / WPS / Keynote / PDF / HTML), Recording mode (2-option segmented: Camera / Camera+Screen), Layout (segmented chips: Close-up / PiP / Side-by-side). All segments 32px tall, 8px radius, active = #1B1F26 bg + 1px #3FB6BD border, inactive = transparent + #2D323B border. Below form: 1 secondary button "Open test deck".
  Card 3 — "Gesture workspace": top = single row toggle "Enable gestures" with 36px switch (off = #2D323B, on = #3FB6BD). Middle = 4 detail lines (recognition state, session, engine, zone). Bottom = 1 secondary button "Calibrate my gestures". Below button: collapsed "Gesture cheatsheet" disclosure — when open shows 4 micro-rows: icon + gesture name + result. Each row 28px tall.
  Card 4 — "Devices & delivery": 1 form row (input device menu), then 4 detail lines (status, device detail, found inputs count, transport). 1 secondary button "Rescan" inline at the bottom right of the form.

  All cards: 12px radius, 1px #232730 border, #14171C bg, 16px inner padding, 12px gap between sections. Each card header: 13px/600 title in #E8E9ED + small 12px/400 hint in #5E636C aligned right (e.g., "Last 5 min", "Auto"). NO card icons in headers — let typography carry hierarchy.

[ Band 3 — Footer · 40px tall, full width, top border 1px #232730 ]
LEFT: version string "v0.5.0 · Director mode" in JetBrains Mono 11px #5E636C.
CENTER: collapsed "Advanced diagnostics" link — chevron rotates 90° on expand. Reveals inline, NOT a modal: shows 4 detail lines (accessibility / chrome automation / scan summary / supported examples) and a row of 4 ghost buttons (Permission settings / Request access / Chrome auth / Refresh).
RIGHT: latency / connection dot (6px teal) + "Connected · 28ms" in 11px JetBrains Mono.

=== INTERACTION & STATES ===
- Hover: surface lifts to #1B1F26, border to #2D323B. 150ms.
- Active button: brightness +0.06.
- Disabled: 40% opacity, no hover, cursor not-allowed.
- Error state on a card: 1px #D4A14A border, small "Why?" link top-right of card.
- Camera disconnected: preview frame shows a single centered state — 32px icon (camera-off) + 14px/500 "No camera connected" + 12px/400 muted hint + 1 secondary button "Reconnect". NO red, NO alarm. Keep it calm.
- Recording active: REC button → square stop icon, REC pill in top bar pulses the dot at 1.2s interval. A thin 2px #D8463E progress bar appears at the very top of the window, growing left-to-right over the last 60s (loop).
- Gesture active: gesture zone rectangle fades from 30% to 100% teal opacity over 200ms, returns to 30% on idle.
- Focus ring: 2px #3FB6BD, 2px offset, visible on Tab navigation only. NEVER on click.

=== EXPLICITLY AVOID ===
- ❌ Gradients on background
- ❌ Glow / blur / glassmorphism on cards (use solid #14171C only)
- ❌ Purple / blue / cyan AI-glow accents
- ❌ Big colorful icons in card headers
- ❌ Emoji
- ❌ Card with "Insight tile" big colored bottom bars
- ❌ Multiple accent colors in one row
- ❌ Animated background, particles, mesh
- ❌ Tailwind-style "primary 500 / 600 / 700" rainbow palette
- ❌ Rounded > 16px on anything
- ❌ Drop shadows in brand color
- ❌ Bouncy / springy micro-interactions
- ❌ Display fonts (no Playfair, no Space Grotesk headlines — keep Inter for everything)

=== FIGMA ORGANIZATION ===
Use Auto Layout everywhere. Build in this order so it's reviewable:
1. Design tokens frame (palette swatches, type ramp, spacing scale, radius scale)
2. Components: StatusPill, PrimaryButton, SecondaryButton, Card, DetailLine, SegmentedChip, FormRow, LiveDot
3. Screen: "Director Console — Default" (camera connected, idle)
4. Screen: "Director Console — Recording" (REC state, recording progress bar)
5. Screen: "Director Console — Camera offline" (disconnected calm state)
6. Screen: "Director Console — Gesture calibration" (modal sheet overlay with 3-step progress)
